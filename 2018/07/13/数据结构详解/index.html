本文参考自严式《数据结构》，对基本数据结构与算法进行简要梳理，方便查阅参考，诚然，编程与算法的学习离不开动手实践，了解基本原理后，一起愉快地撸代码吧

##0x01 数据结构与算法

1. 数据结构

  • 基本概念

    • 数据
      所有能输入到计算机中并能被其处理的用来描述客观事物的符号
    • 数据元素
      数据的基本单位，也称结点(node)或记录(record/entry)。它是数据结构研究的基本对象，在算法/程序中通常把它作为一个整体来考虑和操作
    • 数据项
      有独立含义的数据最小单位，也称域(field)，一个数据元素可由若干个数据项构成
    • 数据对象
      具有相同性质的数据元素的集合
    • 数据类型
      数据值的集合和在这个值集上的一组操作/运算的总称，运算的总称
    • 抽象数据类型（ADT）
      基于一种 逻辑关系的数学模型以及定义在这个模型上的一组操作的数学模型以及定义在这个模型上的一组操作/ 运算

  • 数据结构

    • 个体定义
      相互之间存在一种或多种特定“关系”的数据元素的集合

    • 学科定义
      数据结构是一门研究非数值计算的程序设计问题中计算机操作对象、其间的关系及对之进行基本操作的学科

    • 顺序存储结构
      数据元素之间的逻辑关系通过地址得到直接反映

    • 链式存储结构
      数据元素之间的逻辑关系通过指针得到间接反映

2. 算法及分析

  • 算法特征
    • 有穷性
      一个算法总是在执行有限步后结束，且每步均在有限时间内完成
    • 确定性
      算法中的每条指令必须有确切的含义，不能产生二义性
    • 可行性
      算法中的运算均可通过已经实现的基本运算执行有限次来完成
    • 输入
      一个算法有一个或多个输入
    • 输出
      一个算法应有一个或多个输出，且与输入有特定关系
  • 算法描述
    • 自然语言描述
    • 程序流程图描述
    • 计算机编程语言描述
    • pseudo-code方法
  • 设计要求
    • 正确性(correctness)
    • 健壮性/鲁棒性(robustness)
    • 效率(efficiency)
    • 可读性(readability)

  • 算法分析

    • 空间复杂性(Space Complexity)
      当问题的规模以某种单位由当问题的规模以某种单位由1增至n时，解决该问题的算法实现所占用的空间也以某种单位由时，解决该问题的算法实现所占用的空间也以某种单位由1增至f(n), 则称该算法的空间代价是f(n), 记作S(n) = O(f(n))

    • 时间复杂性(Time Complexity)
      算法中基本操作重复执行的次数依据算法中最大语句频度来估算来估算,它是问题规模 它是问题规模n的某个函数f(n), 算法的时间量度记作T(n) = O(f(n))

##0x02 线性表

1. 概念术语

  • 结构特征
    • 存在唯一的一个被称作“第一个”的数据元素
    • 存在唯一的一个被称作“最后一个”的数据元素 的数据元素
    • 除第一个外，集合中的每个数据元素均只有一个前驱(predecessor)
    • 除最后一个外，集合中的每个数据元素均只有一个后继(successor）

2. 顺序表示及实现

  • 表示及定义
    • 定义
      用一组地址空间连续的(contiguous)存储单元存放的一个线性表
    • 元素地址计算
    • 特点
      逻辑上相邻的数据元素，其物理地址亦相邻；随机存取
    • 表示
      用C/C++ 语言中的一维数组(array)
  • 操作及运算
    • 插入运算

    • 删除运算

  • 优缺点
    • 优点
      • 可随机存取表中任一数据元素
      • 操作直观
        • 缺点
      • 插入、删除运算需要移动大量的元素(on average: about n/2)
      • 预先分配空间需按最大空间分配，RAM利用不充分 利用不充分
      • 逻辑上相邻的数据元素，物理亦要求相邻

        3. 链式表示及实现

  • 特征

    • 用一组地址离散的存储单元存储线性表的数据元素
    • 利用指针实现了用不相邻的存储单元存放逻辑上相邻的数据元素
    • 每个数据元素ai，除存储本身信息外，还需存储其后继的信息，这时的数据元素可看作“ 结点”

  • 单链表

    • 表示
    • 运算
      • 查找
      • 插入
      • 删除
      • 建立
    • 优缺点
      • 它是一种动态存储结构，不需预先分配空间
      • 插入、删除运算无需移动大量的数据元素，节省了时间开销
      • 指针占用额外存储空间
      • 顺序存取数据元素( 不能随机存取) ，查找速度慢

  • 循环链表

  • 双向链表

    • 特征
    • 运算
      • 删除
      • 插入
